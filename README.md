## 第一章：开机流程

你按下电源键，交互平板依次显示蓝色鸿合交互平板启动画面，黑色 UEFI[^1] 启动画面，Minecraft 主题的 GRUB[^2] 引导画面，自检日志和看门狗日志[^3] [^4] 滚屏，Archlinux 主题的 KDE 桌面加载画面[^5]，然后进入桌面。

没什么需要注意操作的地方，场面比较花里胡哨，不过不需要担心。这些画面几乎都可以去掉，但是得要你懂得linux来折腾才行。

> `注释`：
>
> [^1]: **UEFI** (Unified Extensible Firmware Interface) —— 统一可扩展固件接口，替代传统 BIOS 的现代固件标准。负责开机后最基本的硬件初始化、启动设备枚举，并将控制权交给引导加载程序。相较于 BIOS，UEFI 支持 GPT 分区表、Secure Boot 和更快的启动速度。本机固件开启 UEFI 模式，禁用 CSM（兼容性支持模块）。
>
> [^2]: **GRUB** (GRand Unified Bootloader) —— Linux 下最常用的引导加载程序。负责选择要启动的操作系统内核，将其加载到内存后移交控制权。本机的 GRUB 主题已被替换为 Minecraft 风格，纯粹为了好看，不影响功能。
>
> [^3]: **自检日志** —— 内核在启动过程中输出的诊断信息，通过 `printk` 写入内核环形缓冲区。内容包括硬件检测、驱动加载、服务启停等状态。若启动异常，这些日志是定位问题的第一手资料。系统启动后可通过 `dmesg` 命令或 `journalctl -k` 查阅。
>
> [^4]: **看门狗 (Watchdog)** —— 一种硬件或软件定时器机制，用于检测系统是否发生死锁或挂起。若看门狗在设定时间内未收到「喂狗」信号（即系统正常运行的确认），将强制复位系统。本机主板 UEFI 固件中内置硬件看门狗，Linux 内核通过 `iTCO_wdt` 驱动提供支持。
>
> [^5]: **KDE Plasma 启动画面 (Splash Screen)** —— KDE Plasma 桌面环境在加载过程中显示的动画画面，提示桌面核心组件（Plasma Shell、KWin 窗口管理器、Widgets 等）正在初始化。启动完成后自动消失。本机使用了 Archlinux 风格主题的启动画面，好看吧～

## 第二章：关机流程

### 为什么实体键不能用了

机器右下角的实体关机键按下后会卡住，大约 5 分钟后自动断电。

原因是这个按键本来不是直接断电，而是发出一个信号告诉操作系统"用户想关机了"。Windows 下鸿合白板注册了自己的驱动钩子来拦截这个信号，执行关机的整套流程。Linux 没有实现这套钩子，交互平板把信号发出去了，但系统没法接收处理，于是就卡住了。

5 分钟后自动断电是因为主板上的硬件看门狗（Watchdog）——一个独立计时芯片，系统正常时它会定期被重置（"喂狗"），如果超时未收到喂狗信号，就认为系统已死机，强制复位。

### 正确的关机方式

以下三种方法都可以关掉这台机器。

**方法一：通过 KDE 开始菜单关机，不多讲**

**方法二：通过终端(TTY)命令关机**
不想用鼠标的时候，一条命令就能关机。
1. 按 `Ctrl + Alt + T` 打开终端（黑色半透明窗口）
2. 输入以下命令后按回车：
   ```bash
   sudo poweroff
   ```
    或者用[^3]（你没看错）
    ```bash
    fuck
    ```
3. 如果提示输入密码，输入密码后按回车（注意：输入时屏幕不会显示任何字符，连 `*` 都没有，这是 Linux 的安全机制[^2]，正常输入后回车即可）。

**方法三：切换 TTY 强制关机（桌面卡死时使用）**

桌面程序卡死、鼠标点不动时，用这个方法绕过图形界面。

1. 按 `Ctrl + Alt + F3`（或 F1~F10 中的任意一个，只要显示黑屏幕加英文字符就可以使用）
2. 屏幕变为纯黑底白字的控制台界面
3. 在 `login:` 提示后输入**用户名**(k2305)，按回车
4. 在 `Password:` 提示后输入你的**登录密码**(2305)（不会显示字符），按回车
5. 登录成功后输入并按回车：
   ```
   sudo poweroff
   ```
   fuck[^3]指令此处依然有效
6. 屏幕出现日志滚动，然后断电

> `注释`：
>
> [^1]: **TTY（Teletype）** —— TTY 是 Linux 的底层控制台，直接运行在内核之上。它的功能和 Windows 的 cmd 或 PowerShell 有本质区别：cmd 只是 Windows 图形桌面里的一个应用程序，可以随时关掉；TTY 是 Linux 系统的原生交互接口，图形界面（KDE Plasma）反而是运行在 TTY 之上的一个应用程序。Linux 本质上是一个没有图形界面的操作系统，TTY 是它最根本的操作方式。原则上 Linux 可以完全摆脱图形化环境完成日常任务——大量软件被设计为 TUI（Text-based User Interface，基于文本的用户界面，如 btop、nvim、yazi），这是 Linux 的设计逻辑和用户群体决定的。这也是为什么 Linux 提供多个 TTY（一般有6个：F1~F6，但我们班上这台是服务器版，有更多）而 Windows 没有这个概念——因为 Windows 从设计上就是图形操作系统，而 Linux 从一开始就是多用户多任务命令行系统。
>
> [^2]: **Linux 密码输入机制** ——  **`Linux 终端在输入密码时不会显示任何字符`** ，连 `*` 号都没有。这是安全设计：不显示任何内容可以隐藏密码长度信息，防止旁边的人通过数星号个数推断密码位数。这种机制在 `sudo`、`su`、`login`、`ssh` 等所有需要密码的场景下都一致。输入完成后直接按回车即可。
>
> [^3]: **`fuck` 指令** —— 这是编者预设的 shell 别名。当你输入 `fuck` 时，shell 在执行前将其替换为 `sudo poweroff`，其中 `sudo` 用于提权（使普通用户获得 root 权限），`poweroff` 向 systemd 发起关机请求，systemd 再通过内核操作电源管理硬件断电。这里纯粹是无聊瞎搞的。

## 第三章：软件管理

本章是最重要的内容，请详细阅读。

### 为什么不能运行 .exe？

原则上，同为x86_64架构，其指令集应该是相同的，但是：

Windows 的可执行文件以 `.exe` 为扩展名，采用 **PE（Portable Executable）格式**。Linux 的可执行文件采用 **ELF（Executable and Linkable Format）格式**。两者是各自操作系统规定的可执行文件标准，从文件头部结构、段表布局到加载方式都不同。

具体来说：当操作系统运行一个程序时，加载器会先读取文件头部信息，解析出代码段、数据段、导入表、导出表等结构，然后将代码载入内存并跳转到入口点执行。PE 加载器和 ELF 加载器各自只会识别自己的格式。

#### 应该运行什么？

**1. `.deb` 包**
Debian 的标准安装格式。双击用 Discover 安装，或终端执行：
```bash
sudo apt install -f 安装包路径(这一段可以直接手动把文件拖进终端)
```

**2. AppImage**
单文件免安装，下载后赋予执行权限就能跑：
```bash
chmod +x 软件.AppImage
./软件.AppImage
```

**3. Linux x86_64 二进制**
部分软件直接提供编译好的 Linux 可执行程序，没有依赖管理，下载即用。

#### 安装 LinuxQQ 实战

1. 前往 [QQ Linux 版官网 https://im.qq.com/linuxqq ](https://im.qq.com/linuxqq) 下载安装包，选择 **x86_64 架构、deb 派系**。
2. 按 `Ctrl + Alt + T` 打开终端，输入以下命令（将安装包文件拖入终端窗口，路径会自动填入）：
   ```bash
   sudo apt install -f（此处拖入安装包文件或者手动写好文件路径）
   ```
   例如：
   ```bash
   sudo apt install -f /home/k2305/Desktop/linuxqq.deb
   ```
3. 出现以下提示时：
   ```
   确定要安装吗？[Y/n]
   ```
   直接按回车（默认 Y），或输入 `y` 再回车。
4. 等待日志滚动完成，无错误即安装成功。
5. **如果遇到错误**，将终端输出的所有文字复制下来发给 AI，参考以下提问模板：
   ```

   （粘贴错误信息）
   ```
6. 卸载命令：`sudo apt remove linuxqq`

### 我一定要运行 .exe 怎么办？

你可以使用兼容层 **Wine**[^3]。已经在本机上安装配置好了。

#### Wine 是什么？

Wine（Wine Is Not an Emulator）是一个兼容层，它在 Linux 上重新实现了 Windows 软件运行时需要的系统接口，让部分 Windows 程序能够直接运行。

Wine 和虚拟机不同，虚拟机在 Linux 上完整运行一个 Windows 系统，性能开销大；Wine 不模拟 Windows 内核，而是把 Windows 程序的 API 调用翻译成 Linux 系统调用，性能损失小得多，有时候 Linux 上通过 Wine 运行 exe 速度甚至比 Windows 更快，因为 Linux 系统优化更好。

#### 怎么用？

**方法一：命令行运行**
```bash
wine 程序.exe
```
或者打开 Windows 的资源管理器，在资源管理器里面直接双击exe文件：
```bash
wine explorer
```

**方法二：直接双击 .exe 文件**
在文件管理器里找到 .exe 文件，双击即可（如果提示要选择打开方式，就找到 `/usr/bin/wine` 选中）。双击启动需要等待一段时间，因为 Wine 启动要预热。

#### Wine 配置不要乱动，牵一发而动全身

Wine 的配置涉及模拟的 Windows 版本、DLL 覆盖、注册表设置、库依赖链等。当前本机的 Wine 已经过调试，配置了能支撑日常教学软件的环境。如果 Wine 配置被破坏，这台机器可能无法正常进行教学工作（微信、WPS 都是通过 Wine 运行起来的）。

Wine 的配置非常复杂（甚至是社区津津乐道的一个话题），编者也无法在学校调试好一个 Wine，而是从在家里使用、调优了数年的 Wine 配置复制过来的。

**不要做以下操作：**
- 不要运行 `winecfg` 乱改设置
- 不要删除或修改 `~/.wine` 目录下的文件
- 不要手动添加或替换 DLL

> `注释`：
>
> [^1]: **`.deb` 包** —— Debian 软件包格式，本质是一个 ar 归档文件，内含控制信息（依赖、描述等）和安装数据。使用 `dpkg` 处理底层安装，`apt` 自动解析并补全依赖。
>
> [^2]: **AppImage** —— 一种 Linux 便携式软件格式，将程序及其依赖打包为单个文件，无需安装。原理是将 AppImage 挂载为 FUSE 文件系统后执行入口程序。优点是分发简单、不影响系统包管理。
>
> [^3]: **LinuxQQNT** —— QQ 的 Linux 官方重构版，基于 QQ NT 架构（Electron + React），不再依赖 Wine 兼容层。目前支持 x86_64 架构，提供 `.deb` 和 `.AppImage` 两种格式。相比 Wine 版 QQ，原生版在 Linux 下有更好的性能和系统集成度。
>
> [^4]: **APT 与彩蛋** —— `apt` 是 Debian 的包管理前端，负责从软件源获取、安装、升级、移除软件包。输入 `apt moo` 会显示一头 ASCII 艺术奶牛，输入 `apt-get moo` 也是。这是 Debian 社区的一个经典彩蛋，`"本 APT 具有超级牛力"即源于此`。
>
> [^5]: **双击运行 .exe 的原理** —— Linux 文件管理器本身不识别 .exe。系统通过文件关联机制将 `.exe` 扩展名映射到 Wine 启动器（`/usr/bin/wine`）。双击时文件管理器调用 `wine xxx.exe`，Wine 读取 PE 文件头，初始化模拟的 Windows 环境（加载所需 DLL、注册表），然后跳转到程序入口点执行。预热时间主要是 Wine 服务器（`wineserver`）启动和前缀初始化的开销。

## 第四章：文件管理

### Linux 没有分盘机制

Windows 有 C 盘、D 盘、E 盘，每个盘符对应一个独立的磁盘分区。Linux 没有这个设计——它使用**一棵统一的目录树**，所有存储设备都挂载到根目录 `/` 下的某个位置。

具体来说：在 Windows 里你插入一块硬盘，系统给它分配一个新盘符（如 F:），这块硬盘的文件就放在 F 盘里。在 Linux 里你插入一块硬盘，你需要把它"挂载"到某个文件夹上（如 `/mnt/data`），然后访问这个文件夹就是访问这块硬盘的内容。这是 Linux 哲学（一切皆文件）的一部分。

这台机器的文件结构：

| 目录 | 用途 |
|------|------|
| `/` | 根目录，所有文件和目录的起点 |
| `/home/` | 用户数据目录，相当于 Windows 的 `C:\Users` |
| `/home/k2305/` | 你的个人文件夹，桌面、下载、文档都在这里 |
|`/run/media/k2305` | 一般来说，U盘会被挂载到这里|
| `/boot/` | 系统启动文件（不要动） |
| `/usr/` | 系统软件和库（不要动） |
| `/etc/` | 系统配置文件（不要动） |

> 只需要关心 `/home/k2305/Desktop` 下面的内容，其他地方都是为了支撑系统运行的，不需要也不应该去动。

### U 盘挂载

Linux 不会像 Windows 那样插入 U 盘后自动分配一个盘符。不过 KDE Plasma 已经简化了这个过程，和 Windows 上面区别不大了。

> [!WARNING]
> 不安全的拔出可能会让 U 盘里的文件变成乱码或无法打开。特别是老师在上完课要带走课件时，务必先安全移除再拔。

### 不要乱动系统文件

这台机器上运行着大量由编者编写的脚本和服务，用于适配教学环境（自动挂载、网络配置、软件兼容等）。

**除桌面目录外，不要修改或删除任何文件，除非你清楚自己在做什么。** 误删一个脚本可能导致白板无法工作、网络连不上、甚至系统无法正常开关机。

### 文件整理规范

这台机器没有专人维护，日常使用中生成的文件请自行整理。

- 临时添加的文本文件，文件名建议包含**日期编号**，格式为 `DDMMYY`，例如 `260622`（2026 年 6 月 22 日）
- 课堂结束后，将当堂文件移动到对应科目的文件夹

> 不要使用任何所谓的"电脑管家"或"清理大师"类软件。Linux 的文件系统和 Windows 完全不同，这类软件不仅用不上，还可能破坏系统权限和脚本依赖。磁盘空间清理请使用 **Filelight**（已预装），在 KDE 开始菜单中可找到，可以方便快捷通过色块判断文件大小与类型，直接清理文件。同时推荐基于 TUI 的 `ncdu`，功能相同，在终端执行 `ncdu` 就可以打开。


### 桌面图标异常

如果机器因断电等异常关机，下次开机后桌面文件图标可能会排列混乱、位置错乱。这是 KDE Plasma 桌面配置文件因非正常关机未能保存所致。

解决方法：手动拖动调整桌面图标位置，排列到习惯的顺序后，再**正常关机一次**（通过开始菜单关机，不要断电）。下次开机桌面布局就会固定下来。

> `注释`：
>
> [^1]: **Linux 文件树** —— Linux 使用单根目录树结构，所有文件从 `/` 开始向下展开。无论系统有几个硬盘、几个分区，最终都映射到这一棵树上，不会产生 Windows 那种多个盘符根目录。
>
> [^2]: **除非你知道自己在做什么** —— 这是软件社区的一句经典告诫。关键目录下的文件变动可能直接导致软件配置出错、系统无法启动、桌面无法加载、网络服务中断。本条机器的部分目录还挂载了编者编写的自动化脚本和定时任务，误删或误改后不会自动恢复。对于任何文件，只有深刻理解文件作用才能修改。
>
> [^3]: **主要系统目录说明** ——
> - `/usr/` —— 系统软件和库文件，相当于 Windows 的 `C:\Program Files` 和 `C:\Windows\System32` 的结合
> - `/bin/` 和 `/sbin/` —— 存放系统启动和运行时必需的可执行程序。`/bin/` 是普通命令（如 `ls`、`cp`），`/sbin/` 是系统管理命令（如 `fdisk`、`mkfs`）。现代 Linux 中 `/bin` 和 `/sbin` 通常是 `/usr/bin` 和 `/usr/sbin` 的符号链接
> - `/proc/` —— 一个虚拟文件系统，内核运行时动态生成，反映进程、硬件、内核参数等运行时信息。里面的文件不在磁盘上，占用空间为零
> - `/run/` —— 存放系统运行时产生的临时文件，如进程 PID 文件、套接字等。系统重启后自动清空
> - `/mnt/` —— 临时挂载点的通用目录。插入 U 盘或移动硬盘时，KDE 通常挂载到 `/run/media/` 而不是 `/mnt/`

## 第五章：网络问题

### 怎么这么多网卡？

第一次打开 KDE 网络设置的时候我被吓到了，一台教室一体机装了 4 个网卡。

在 KDE 系统托盘中点击网络图标，会列出所有可用网络。一般来说有线连接最稳定、速度最快，优先连它。只需要选择**能上网的那个**连上就行，其他网卡不用管，系统会自动路由流量到已连接的接口。

### Clash Verge

系统右下角系统托盘里有一个小猫咪图标，那是 **Clash Verge**——一个网络分流代理工具。Clash 是一个通用的网络代理内核，支持根据规则自动分流流量（比如国内的网站直连，国外的网站走代理）。

这台机器的网络环境比较特殊——**系统手动设置了代理网络**，所有网络连接都要经过 Clash 分流才能正常出入机器。即使你不需要翻墙，Clash 也必须运行在**直连模式**下作为网络流量的通道。

编者临走时已将其切换为**直连模式**，所有流量直接通过，不影响正常上网速度。需要翻墙的资源编者已经部署好了，日常教学完全不需要动它。

- 如果你需要特殊的 VPN 上网方式，自行搜索学习clashverge的教程,或者联系编者。
- **不要关掉 Clash Verge**，关掉后整个系统会失去网络连接

> [!WARNING]
> Clash Verge 不能关闭。这不是可选软件，是系统网络基础设施的一部分。

### 网络托盘图标不见了

如果右下角的网络图标突然消失且无法连接网络，通常是 NetworkManager 服务异常或未随系统启动。打开终端执行：

```bash
sudo systemctl restart NetworkManager
sudo systemctl enable NetworkManager
```

第一条命令立即重启网络服务，第二条命令确保 NetworkManager 在下次开机时自动启动。执行完成后重启电脑，托盘图标和网络功能应该可以恢复正常。

> `注释`：
>
> [^1]: **网卡** —— 网络接口控制器，计算机连接网络的硬件设备。每个网卡有一个唯一的 MAC 地址和系统分配的网络接口名（如 `eth0`、`wlan0`、`enp2s0`）。Linux 通过接口名而非盘符来区分不同的网络设备。
>
> [^2]: **Clash 与 Clash Verge** —— Clash 是一个基于规则匹配的通用网络代理内核，使用 Go 语言编写，支持多种代理协议（Shadowsocks、VMess、Trojan、Socks5、HTTP 等）。Clash Verge 是 Clash 的一个图形化前端，提供界面化的规则管理和代理切换功能，底层仍由 Clash 内核处理网络流量。
>
> [^3]: **Clash 分流原理** —— Clash 根据用户定义的规则集，对每个网络请求判断其目标地址属于"直连""代理"还是"拒绝"。规则匹配顺序从上到下，命中即停止。例如：国内 IP 段 → 直连，被屏蔽域名 → 走代理，其余 → 直连。这台机器由于系统代理被手动设置，所有流量必须经过 Clash 的虚拟网卡进出，即使规则全是直连也不能关闭 Clash。
>
> [^4]: **VPN** —— 虚拟专用网络，通过加密隧道将设备连接到远程网络，常用于，嗯，你懂。
>
> [^5]: **NetworkManager 与 Linux 网络管理方式** —— Linux 传统的网络管理方式是通过 `/etc/network/interfaces` 或 `ip` 命令手动配置。NetworkManager 是一个更高层的守护进程，自动检测网络硬件变化、管理连接配置、通过 D-Bus 与桌面环境（KDE Plasma）通信，实现系统托盘网络图标、Wi-Fi 扫描、自动重连等功能。本机使用 NetworkManager 管理所有网络连接。
>
> [^6]: **系统服务（systemd）** —— systemd 是 Linux 的系统和服务管理器，PID 为 1，负责启动、停止、监控系统服务。`systemctl` 是其管理命令：`restart` 重启服务，`enable` 设置开机自启，`status` 查看服务状态。NetworkManager 就是由 systemd 托管的一个系统服务。

## 第六章：快捷操作

和 Windows 不一样的操作习惯，这里列出来。

### 触控板手势

这台一体机支持触控板多点手势，和 Windows 的触摸板操作逻辑类似：

- **三指上划** — 显示所有正在运行的后台应用（任务视图），类似 Windows 的任务视图
- **三指下滑** — 在任务视图模式下，选中一个应用窗口并按住三指向下滑，可以关闭该应用
- **三指左右划** — 在不同工作区之间切换。Linux 原生支持多工作区（虚拟桌面），相当于把屏幕空间扩展到多个桌面，每个工作区可以放不同的窗口，互不干扰
- **单指从屏幕最顶端滑行到最下方** — 锁屏

### 触屏操作

这台一体机的屏幕是触摸屏，操作逻辑和安卓手机类似，和 Windows 不同：

- **单击** — 打开文件或启动程序
- **长按** — 进入编辑模式（重命名、选中文本等）

这**和 Windows 的触屏逻辑不同**，但这才是触屏该有的机制，请多适应。小心点多了会打开很多窗口，记得提醒老师。

- **按住 Win 键 + 鼠标左键拖动** — 移动当前窗口位置
- **按住 Win 键 + 鼠标右键拖动** — 调整当前窗口大小

这是 KDE Plasma 的一个特色功能：按住 Win 键后，鼠标左键和右键分别变为移动窗口和缩放窗口的操作，不需要精确点到窗口标题栏或边框。

### 常用快捷键

|------|--------|------|
| 打开终端 | `Ctrl + Alt + T` | 快捷打开终端 |
| 锁屏 | `Win + L` | 和 Windows 一样 |
| 切换窗口 | `Alt + Tab` | 和 Windows 一样 |

## 第七章：愉快的踩坑排错记录

这一章记录了一些实际遇到过的问题和解决方法。如果以后遇到同样的情况，可以按步骤排查。

### 插入U盘没有反应？

插入U盘后右下角没有弹出提示，文件管理器侧边栏也看不到 U 盘。按以下顺序排查：

**第一步：重新插拔**

先拔出来换个口再插一次。如果右下角弹出提示了就搞定了，没有的话看下一步。

**第二步：用 `lsblk` 看系统有没有识别到设备**

打开终端，输入：

```bash
lsblk
```

回车后会看到类似这样的输出：

```bash
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
sda      8:0    0 238.5G  0 disk 
├─sda1   8:1    0   512M  0 part /boot
└─sda2   8:2    0   238G  0 part /
sdb      8:16   1  14.5G  0 disk 
└─sdb1   8:17   1  14.5G  0 part 
```

重点看有没有 `sdb`（或者 `sdc`、`sdd`，取决于你插了几个盘）。`sda` 一般是系统盘，`sdb` 往后才是 U 盘。[^1]

- 如果能看到 `sdb` 但没有 `MOUNTPOINTS`（没有挂载点），说明系统识别到了 U 盘但没有自动挂载。说明文件系统不支持，把文件系统类型记下来（比如 NTFS、exFAT）[^2]，然后问 AI 怎么装对应的驱动。
- 如果 `lsblk` 输出里完全没有 `sdb` 之类的新设备，说明系统根本没识别到硬件，看下一步。

**第三步：用 `lsusb` 确认是不是硬件问题**

在同一终端里输入：

```
lsusb
```

输出类似：

```
Bus 001 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub
Bus 001 Device 002: ID 0bda:0129 Realtek Semiconductor Corp. RTS5129 Card Reader Controller
Bus 002 Device 001: ID 1d6b:0003 Linux Foundation 3.0 root hub
```

记住现在的输出，然后**拔掉 U 盘**，再跑一次 `lsusb`，对比两次输出：

- 如果两次输出**不一样**（拔掉后少了一个设备），说明 USB 口能识别到 U 盘，但系统加载不了驱动。把两次的输出发给 AI，让它帮你分析缺什么驱动。
- 如果两次输出**一模一样**（插不插都没区别），说明 USB 口根本没检测到 U 盘，硬件层面就没反应。直接看下一步。

**第四步：物理层面的问题（饺子醋来咯）**[^3]

如果以上都排除了，大概率是接触不良。

拿一小团卫生纸塞进 USB 口，把 U 盘的金手指垫高一点再插（真有过不止一回是这么解决的）。神奇吧。

> **注释：**
>
> [^1]: `sd*` 是 Linux 对 SCSI/SATA/USB 存储设备的命名方式。`sd` 即 SCSI Disk，后面的字母按检测顺序分配——`sda` 是第一个、`sdb` 第二个，以此类推。数字表示分区，比如 `sdb1` 就是 U 盘的第一个分区。
>
> [^2]: 文件系统决定了 U 盘怎么存文件。Windows 常用 FAT32（兼容最好但单文件不能超 4G）、NTFS（支持大文件但 Linux 需额外装 `ntfs-3g`）、exFAT（适合大文件跨平台，需装 `exfatprogs`）。在 Linux 上可以用 `sudo file -s /dev/sdb1` 查看 U 盘的文件系统类型。
>
> [^3]: 对，这一部分内容就是冲着这一部分来的哈哈哈哈。


### KDE 桌面卡住不动了

鼠标点不动、键盘没反应、画面定格，但主机还在运行（风扇在转、电源灯亮）。不用急着按电源键强关。

**第一步：切到另一个 TTY**

按 `Ctrl + Alt + F3`（如果没反应就试 F4、F5……直到屏幕变成黑底白字的登录界面）。你现在在一个干净的控制台上了。

**第二步：登录**

输入用户名 `k2305` → 回车 → 输入密码`2305`(不要看屏幕） → 回车。

**第三步：切回 KDE 所在的 TTY，确认卡死了，记住那个TTY的编号**

**第四步：杀掉 KDE 会话**

假如知道了 KDE 在 TTY1，在新的 TTY 执行：

```bash
sudo ps -t tty1
```

你会看到 TTY1 上跑了一堆进程，最上面那个是父进程。
示例：
```bash
 TTY          TIME CMD
    643 tty1     00:00:00 bash
    693 tty1     00:00:00 startplasma-wayland
    776 tty1     00:00:06 Xwayland
   1060 tty1     00:43:07 flclash
   1201 tty1     00:02:32 FlClashCore
   1374 tty1     00:12:25 firefox
   1499 tty1     00:00:00 forkserver
   1516 tty1     00:00:00 Socket Process

```
如果输出太长，看不到顶部，可以执行：
```bash
sudo ps -t tty1 | more 
```
使用`q` 退出阅读模式

目标通常是 `bash`或者`startplasma-wayland`。记下他们的 PID，然后：

```bash
sudo kill -9 <PID>
# 比如
sudo kill -9 643 
sudo kill -9 693
```

**第五步：启动血浆桌面**[^4]

在当前 TTY 输入 `startplasma-wayland`（输入 `startpla` 后按 Tab 可以自动补全）。屏幕切回图形界面，KDE 重新加载。

**第六步：还不行就重启**

按第二章的方法重启。别再挣扎了。

> `注释`：
>
> [^4]: **血浆桌面** —— `plasma` 本意是"血浆"或"等离子体"，KDE Plasma 的名字取了后一个意思。这里故意翻译成"血浆桌面"是个内部玩笑。

## 第八章：体验 TUI！

TUI（Text-based User Interface）就是在终端里跑的有界面的程序。没有花里胡哨的图形，全靠字符和颜色，跑得快、省资源。

本章介绍本机预装的一些实用 TUI 工具。打开终端的方法还记得吧？`Ctrl + Alt + T`。

### btop —— 系统监控

`btop` 算是终端版的资源管理器，比 Windows 的任务管理器好看多了。

在终端输入 `btop` 回车：

```bash
btop
```

界面分为几个区域：
- **左上**：CPU 每个核心的占用率和温度
- **右上**：内存和磁盘占用
- **中下**：进程列表，跟任务管理器一样

操作方式：
- 鼠标可以直接点（如果终端支持）
- 按 `1` 切换 CPU 视图
- 按 `m` 排序进程（按内存/CPU）
- 按 `q` 退出

> TUI 程序有一个共同点：**大部分功能都标在界面顶栏或底栏上了**，一眼就能看到按什么键。

### yazi —— 终端文件管理器

`yazi` 是一个用 Rust 写的文件管理器，可以在终端里浏览文件、预览内容，批量重命名，批量操作文件。

启动：

```bash
yazi
```

基本操作：

| 按键 | 作用 |
|------|------|
| `hjkl` | 移动光标（和 vim 一样） |
| `Enter` | 打开文件或进入目录 |
| `~` | 回到 home 目录 |
| `y` | 复制文件 |
| `p` | 粘贴文件 |
| `d` | 剪切文件 |
| `dd` | 标记删除 |
| `空格` | 多选 |
| `q` | 退出 |

右侧会显示文件的预览——图片、文本、目录结构都能直接看，不用打开另一个程序。yazi 强大在于快速和集成，功能远不止这些（还支持插件），但学习曲线陡峭，没有兴趣可以不去使用。

### nvim —— 编者最爱用，也是真在编写这份文档时用的编辑器

`nvim`（Neovim）是一个终端文本编辑器，主要面向代码编写。本机已经配置好了外观、语法高亮、补全、AI助手等功能。

在 Linux 社区里，编辑器是个很严肃的话题。Vim/Neovim 和 Emacs 之间的"编辑器之战"持续了几十年，至今没有结果。Vim 的设计核心是**模式编辑**——你不在编辑器里直接打字，而是在"普通模式"下用快捷键操控文本，按 `i` 进入"插入模式"打字，按 `Esc` 回到普通模式。这套逻辑初看反直觉，但上手后编辑效率远超普通编辑器。(vscode太重了，和集成开发环境坐一桌去）

这也导致了一个经典网络 meme——"如何在 Vim 里退出"。新手打开 Vim 发现打不了字也退不出去，只能满互联网搜怎么关掉它。现在你知道了：按 `Esc` 然后 `:q!` 回车。

启动：

```bash
nvim 文件名
```

比如打开这个手册：

```bash
nvim ~/Documents/五班电教手册.md
```

> [!WARNING]
> nvim 的学习曲线非常陡峭。如果不小心卡住了，按 `Esc` → `:q!` → 回车就能退出。

### neofetch —— 系统信息展示

`neofetch` 会在终端里输出系统信息——内核版本、桌面环境、CPU、内存、显卡、分辨率等，左边配一个 Debian 的 ASCII 艺术 logo。

启动：

```bash
neofetch
```

纯展示用，用来列出自己的系统信息，在 Linux 社区里截图发帖时，neofetch 是标配。

> 注意：neofetch 已经停止维护，社区正在迁移到它的 Rust 替代品 `fastfetch`。

### ncdu —— 磁盘空间分析

如果你觉得电脑空间不够了，用 `ncdu` 可以快速看到底是哪些文件占的空间。

启动：

```bash
ncdu
```

它会扫描当前目录并显示一个按大小排序的文件列表。导航用方向键，按 `d` 可以删除选中的文件。

你还是用`Filelight`吧。

### hollywood

`hollywood` 是一个纯粹为了好玩的程序——打开后终端会像黑客电影里那样，多窗口同时滚代码字符，看起来好像在进行什么高深莫测的操作。

启动：

```bash
hollywood
```

要退出按 `Ctrl + C` 或 `Ctrl + D`。

> 这个程序的作者说得很直白："当坏人入侵主角电脑时到底在干什么？——看起来他们就是在跑 hollywood。"

### 其他

终端里还装了这些小玩具，自己试试：

```bash
sl             # 如果你输入了 sl 而不是 ls，会有一辆火车开过
lolcat         # 给文字加上彩虹渐变色
```

用法举例：

```bash
neofetch | lolcat
```

会输出彩虹色的系统信息。
